Mission Briefing - SSH Security

A breach has been detected in your network. The adversary is attempting a brute-force attack to gain unauthorized SSH access to your system. Your mission is to secure the server and stop the attack before the adversary gains control.

Mission Objectives:

  • 1. Review SSH login attempts to identify the attack.
  • 2. Disable root login to prevent unauthorized access.
  • 3. Set up key-based authentication for extra security.
  • 4. Block repeated login attempts using iptables.

Step-by-Step Instructions:

Step 1: Review failed SSH login attempts.

Use this command to check for failed login attempts:

grep 'Failed password' /var/log/auth.log

Step 2: Disable root login for SSH.

Edit the SSH configuration file to disable root login using this command:

nano /etc/ssh/sshd_config

Find the line that says PermitRootLogin and set it to no.

Step 3: Restart the SSH service to apply the changes:

systemctl restart sshd

Step 4: Block the attacker's IP using iptables:

iptables -A INPUT -s [Attacker IP] -j DROP

This command will drop all traffic from the attacker's IP.
